## 1. Introduction

### What is Theory of Computation?
- Study of computational models
- What problems can be solved?
- How efficiently can they be solved?

### Computational Models
1. **Finite Automaton**: Simple memory
2. **Pushdown Automaton**: Stack memory
3. **Turing Machine**: Unlimited memory

### Chomsky Hierarchy

| Type | Grammar | Automaton | Language |
|------|---------|-----------|----------|
| Type 0 | Unrestricted | Turing Machine | Recursively Enumerable |
| Type 1 | Context-Sensitive | Linear Bounded PDA | Context-Sensitive |
| Type 2 | Context-Free | Pushdown Automata | Context-Free |
| Type 3 | Regular | Finite Automata | Regular |

## 2. Finite Automata

### What is Finite Automaton?
- Mathematical model
- States and transitions
- Reads input left to right

### Components
- **Q**: Finite set of states
- **Σ**: Input alphabet
- **δ**: Transition function
- **q₀**: Initial state
- **F**: Final/accepting states

### DFA (Deterministic)
- One transition per input from each state
- **Transition Function**: δ: Q × Σ → Q

### NFA (Non-Deterministic)
- Multiple possible transitions
- **Transition Function**: δ: Q × Σ → P(Q)
- Can have ε-transitions

### DFA vs NFA
- Every NFA can be converted to DFA
- DFA is faster (no backtracking)
- NFA is easier to design

## 3. Regular Languages

### Definition
- Language recognized by Finite Automaton
- Can be described by Regular Expressions

### Regular Expressions
| Symbol | Meaning |
|--------|---------|
| a | Character 'a' |
| ε | Empty string |
| * | Zero or more |
| + | One or more |
| ? | Zero or one |
| \| | Union (or) |
| ( ) | Grouping |

### Operations on Regular Languages
- **Union**: L₁ ∪ L₂
- **Concatenation**: L₁L₂
- **Kleene Star**: L*

### Pumping Lemma (Regular Languages)
- For regular language L
- ∃ pump length n
- For any string w ∈ L with |w| ≥ n
- w = xyz where:
  - |y| ≥ 1
  - xyⁱz ∈ L for all i ≥ 0

### Properties
- Closed under union, intersection, complement
- Decidable: Membership, Emptiness, Finiteness

## 4. Context-Free Languages

### Definition
- Generated by Context-Free Grammar
- Recognized by Pushdown Automaton

### Context-Free Grammar (CFG)
- **Variables/Symbols**: Non-terminals
- **Terminals**: Actual symbols
- **Start Symbol**: Beginning variable
- **Productions**: Rules for derivation

### CFG Example
```
S → aSb | ε
```
Generates: {aⁿbⁿ | n ≥ 0}

### Derivation
- **Leftmost**: Replace leftmost non-terminal first
- **Rightmost**: Replace rightmost non-terminal first

### Parse Tree
- Root: Start symbol
- Leaves: Terminals
- Internal nodes: Non-terminals

### Ambiguity
- String has multiple parse trees
- Inherently ambiguous languages exist

### Pumping Lemma (CFL)
- For CFL L
- ∃ pump length n
- For any string w ∈ L with |w| ≥ n
- w = uvxyz where:
  - |vxy| ≤ n
  - vy ≠ ε
  - uvⁱxyⁱz ∈ L for all i ≥ 0

## 5. Pushdown Automata

### What is PDA?
- Finite Automaton + Stack
- More powerful than FA

### Components
- **Q**: States
- **Σ**: Input alphabet
- **Γ**: Stack alphabet
- **δ**: Transition function
- **q₀**: Initial state
- **Z₀**: Initial stack symbol
- **F**: Final states

### Types
- **Accept by Final State**: End in accepting state
- **Accept by Empty Stack**: Stack becomes empty

### PDA vs FA
- PDA can recognize {aⁿbⁿ | n ≥ 0}
- FA cannot (requires counting)

## 6. Turing Machines

### What is Turing Machine?
- Most powerful computational model
- Infinite tape with read/write head

### Components
- **Q**: Finite states
- **Σ**: Input alphabet
- **Γ**: Tape alphabet (includes □ blank)
- **δ**: Transition function
- **q₀**: Initial state
- **q_accept**: Accepting state
- **q_reject**: Rejecting state

### Transition Function
- δ(q, a) = (p, b, direction)
- p: New state
- b: Symbol to write
- direction: L or R

### Variants
- **Multi-tape TM**: Multiple tapes
- **Non-deterministic TM**: Multiple choices
- **En Turing Machine**: Writes output

### Church-Turing Thesis
- Turing Machine captures notion of computability
- Anything computable can be computed by TM

### Halting Problem
- Cannot decide whether arbitrary TM halts
- Undecidable

## 7. Complexity Theory

### Time Complexity
- **P (Polynomial Time)**: Solvable in O(nᵏ)
- **NP (Non-deterministic Polynomial)**: Solution verifiable in polynomial time
- **NP-Complete**: Hardest problems in NP
- **NP-Hard**: At least as hard as NP-Complete

### P vs NP Problem
- Biggest unsolved problem in CS
- P = NP? (unknown)

### Reducibility
- Problem A ≤ₚ Problem B
- If we can solve B efficiently, we can solve A

### Classic NP-Complete Problems
- **SAT**: Boolean satisfiability
- **Vertex Cover**: Minimum vertices covering edges
- **Hamiltonian Path**: Visit each vertex once
- **Traveling Salesman**: Shortest tour visiting all cities
- **Knapsack**: Maximize value with weight limit
